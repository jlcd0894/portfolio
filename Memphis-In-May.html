Memphis In May

A re-imagined website to create a memorable experience.

Memphis in May is an International festival whose main purpose is to promote and celebrate Memphis culture, foster economic growth, and enhance international awareness through education. They host the city’s largest events like the Beale Street Music Festival and the World Championship Barbecue Cooking Contest. Memphis in May also produces extensive education, international, and economic programs for the city.

Memphis In May

Completed

Role

Project Link

memphisinmay.org

The Mission

Designing and developing a website that encapsules all 4 events of the Memphis In May festival. We were tasked with re-designing an out-of-date website using the most recent web practices and aesthetics, as well as making it intuitive enough for older people to use, the biggest challenge was unifiyng their 4 events under a single banner, while making it easy to edit in case the organizers needed to make changes in the future.

Art direction

For this projects there were a lot of stakeholders involved, so every decision had to be double checked, from the creation of the sitemap and knowing which pages to keep, to the rough sketches. Every single detail of the process from beginning to end was meticulously planned.

Low Fidelity

In order to make a aethetically pleasing website, as well as making it easy to update, we decided to templatize the design, that way a lot of the pages would be similar in layout and structure, varying just the information within them. This made it easier to create the low fidelity wireframes as a lot of the elements were re-used throuhgout the pages.

Style Guide

Even though the events had different logos and colors, we decided to unify every visual aspect in this website under the Memphis In May brand, using their colors, fonts, and elements throughout every single page.

Implementation

Using Memphis In May's guidelines and assets, I designed a very brand focused high fidelity wireframe. As you might be able to see, a lot of the pages look very similar, this is because we tried to make them all follow a cenrtain structure so the layout can be easily updated in the future without much hassle. As you might be able to tell, this wireframe contains more artboards than the low fidelity wireframe, this is because the stakeholders wanted to see as many pages as they could fleshed out before giving the green light to start development, I eneded up having to design 50 plus high fidelity artboards.

Event Focused

As an event organizer, the most important thing for them was to have all 4 events be visible as soon as you enter the page, that's why we created this layout, each event easily identifiable with their respective logos, dates, and a quick way to buy tickets or read more information about them. All you need, above the fold.

Filling The Gaps

One thing the stakeholders were'nt very fond of was all the white space, they felt we could do something with it, so we decided to create an illustration to add as a background throughout the website, subtle enough that it wont be intrusive, but clear enough to know the space is not empty. Using some kind of topography was a suggestion that was discussed as a team, and with the stakeholders that was percieved really well.

Targeted Subscritions

Each event that Memphis In May hosts requires a lot of information, that's why we decided to create a unified newsletter subscription component, were the user can easily chose which event they are interested in, and just receive emails about those selected,

Event Sneak Peeks

As mentioned before, every page has a similar layout and structure to an extent, and in order to make them unique we decided to use specific event assets for each of the pages, the most obvious thing being the videos in the hero sections of each event main page. As soon as you go into any event page you'll be greeted by a hero section containing the name, date, and a video background where you can see the events from previous years, a sneak peek of what you're in for.

Unique Navigation

The navigation for this site was a hard thing to deal with, we have the main website navigation, and each event required it's own navigation containing several pages. We didn't want to mess with the main navigation so we decided to create specific sticky menus for each event, that would only show up once you go into any of them. These menus containg multiple pages, specific to each event, and once the user reaches a certain point they stay sticky to the top, that way the user always has access to it no matter how far they scroll.
